# 小红书内容整理台

一个本地运行的小工具，用来整理小红书收藏、点赞或手动记录的帖子。它可以按主题归类内容，提取核心摘要和要点，并保留原帖跳转链接。

## 使用方式

电脑上可以直接用浏览器打开 `index.html`。手机上更推荐把整个 `xhs-content-organizer` 文件夹放到 GitHub Pages、Netlify、Vercel 或任意 HTTPS 静态站点，这样可以添加到手机主屏幕，像一个小 App 一样打开。

如果只是临时在手机上试用，也可以把文件夹传到手机，再用支持本地 HTML 的浏览器或文件管理器打开 `index.html`。这种方式能用导入、分类、搜索和导出，但“添加到主屏幕”和离线缓存通常需要 HTTPS。

支持导入：

- JSON：数组，或包含 `items` / `notes` / `data` 的对象
- CSV：首行作为字段名
- TXT / Markdown：用空行分隔每篇内容，第一行作为标题，正文里自动识别链接

推荐字段：

```json
{
  "title": "标题",
  "content": "正文或备注",
  "author": "作者",
  "source": "收藏",
  "link": "https://www.xiaohongshu.com/explore/...",
  "tags": ["绘画", "水彩"]
}
```

## 功能

- 自动分类：绘画、书单、语言学习、美食、穿搭、旅行、学习效率、其他
- 可编辑分类规则：修改关键词后会重新归类
- 核心信息提取：摘要、要点、命中关键词、匹配置信度
- 精准查找：按标题、正文、摘要、作者、关键词、链接搜索
- 结果导出：导出包含分类和摘要的 JSON
- 手机界面：底部切换列表、详情、规则
- 本地保存：导入和规则修改会保存到浏览器本地存储
- PWA：部署到 HTTPS 后支持添加到主屏幕和基础离线缓存

## 说明

小红书没有稳定公开的收藏/点赞批量导出接口。这个版本先做“整理与检索层”，数据可以来自手动整理、浏览器保存的链接列表、CSV、JSON，或后续接入登录态采集脚本。
