# 小红书内容整理台

这是一个可以在手机上使用的小工具，用来整理小红书收藏、点赞或复制下来的帖子链接。它会根据帖子标题、正文、链接文案和评论内容自动分类，并提取核心摘要、要点和原帖跳转链接，方便之后快速查找。

## 适合解决什么问题

小红书收藏和点赞多了以后，经常会出现这些情况：

- 想找一篇帖子，但只记得大概内容
- 收藏里混着美食、游戏、电影、学习、旅行等内容
- 同样是美食，有的是烹饪教程，有的是探店攻略
- 同样是娱乐内容，有的是游戏推荐，有的是电影片单
- 原帖链接散落在聊天记录或剪贴板里，不好统一整理

这个工具就是为了把这些内容集中起来，自动分细类，再通过搜索和链接跳转找回去。

## 主要功能

- 手机友好的界面
- 支持上传 `JSON / TXT / Markdown` 文件
- 支持从小红书复制分享链接后粘贴导入
- 上传后自动生成细分类，不需要提前设置分类
- 支持按标题、摘要、作者、关键词、链接搜索
- 自动提取核心摘要、要点、命中关键词
- 保留原帖链接，可以一键跳回小红书原帖
- 数据会保存在当前浏览器本地
- 支持导出整理后的 JSON
- 部署到 GitHub Pages 后，可以添加到手机桌面，像小 App 一样使用

## 自动分类示例

工具会尽量按照“领域 + 用途”来分，而不是只给一个很粗的分类。

例如：

```text
美食·烹饪教程
美食·探店攻略
旅行·旅行攻略
学习·学习计划
绘画·灵感素材
阅读·清单推荐
游戏·游戏推荐
游戏·游戏攻略
影视·片单推荐
影视·影评解析
动漫·动漫推荐
音乐演出·歌单推荐
家居生活·经验避坑
穿搭美妆·测评对比
```

这样“烹饪教程”和“美食探店”不会都挤在“美食”下面；游戏推荐、电影片单也不会被错误放进“教程方法”或“资料收藏”里。

## 手机使用方法

打开 GitHub Pages 页面：

```text
https://你的用户名.github.io/你的仓库名/
```

如果是这个项目当前仓库，可以打开：

```text
https://fjy934825-jpg.github.io/Little-Red-Book-Classifying-Tool/index.html?app=v6
```

打开后可以直接使用。想像 App 一样放到手机桌面：

### 安卓 / 华为手机

1. 用手机浏览器打开 GitHub Pages 页面
2. 确认页面能看到“导入、粘贴、示例、导出”
3. 打开浏览器菜单
4. 选择“添加到桌面”或“添加到主屏幕”

如果桌面版显示旧页面，先删除旧桌面图标，再重新添加。

### iPhone

1. 用 Safari 打开 GitHub Pages 页面
2. 点击分享按钮
3. 选择“添加到主屏幕”
4. 点击“添加”

## 从小红书导入链接

最稳定的方式是复制链接后粘贴导入：

1. 在小红书 App 里打开一篇帖子
2. 点击“分享”
3. 选择“复制链接”
4. 回到这个工具
5. 点击顶部“粘贴”
6. 点击“读剪贴板”，或者长按输入框手动粘贴
7. 点击“加入整理台”

如果手机浏览器不允许自动读取剪贴板，就手动粘贴即可。

## 上传文件导入

也可以点击“导入”，选择一个文本或 JSON 文件。

### TXT / Markdown 格式

每篇帖子之间空一行：

```text
空气炸锅鸡腿低油晚餐教程
鸡腿肉用生抽、蒜末腌20分钟，180度烤15分钟后翻面，适合新手做饭。
https://www.xiaohongshu.com/explore/xxxx

最近看完的高分悬疑电影片单
这几部电影反转很强，适合周末晚上看，附简单观后感和避雷点。
https://www.xiaohongshu.com/explore/yyyy
```

### JSON 格式

推荐字段：

```json
[
  {
    "title": "标题",
    "content": "正文或备注",
    "comment": "评论或补充信息",
    "author": "作者",
    "source": "收藏",
    "link": "https://www.xiaohongshu.com/explore/xxxx",
    "tags": ["游戏", "推荐"]
  }
]
```

其中 `comment`、`comments`、`评论`、`评论内容` 也会参与分类。

## 部署到 GitHub Pages

1. 把这些文件放在仓库最外层：

```text
index.html
manifest.webmanifest
sw.js
icon.svg
README.md
```

2. 打开仓库 `Settings`
3. 进入 `Pages`
4. `Source` 选择 `Deploy from a branch`
5. `Branch` 选择 `main`
6. 文件夹选择 `/root`
7. 保存

等待 1-3 分钟后，GitHub 会生成 Pages 网址。

## 更新版本后手机还是旧页面怎么办

手机浏览器和桌面小 App 可能会缓存旧版本。

可以这样处理：

1. 上传新版文件到 GitHub
2. 等 1-3 分钟
3. 手机打开带版本号的网址，例如：

```text
https://fjy934825-jpg.github.io/Little-Red-Book-Classifying-Tool/index.html?app=v6
```

4. 如果桌面小 App 还是旧的，删除桌面图标
5. 在浏览器里打开新版页面
6. 重新添加到桌面

## 注意事项

- 这个工具不会自动登录或爬取小红书账号内容
- 数据保存在当前浏览器本地，换手机或换浏览器不会自动同步
- 建议定期点击“导出”，保存整理后的 JSON
- 分类是基于关键词和内容规则自动推断，不是 AI 在线识别；如果内容太少，分类可能不够准确
- 链接能否打开取决于小红书原帖是否还存在、是否需要登录、以及平台跳转限制
